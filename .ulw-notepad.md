### World map research (bg_a3fa9b51) — RETURNED
- **Map source DECISION**: **Wikimedia Simple World Map** — https://commons.wikimedia.org/wiki/Special:FilePath/Simple_world_map.svg — **21KB, CC0 public domain**, equirectangular, angular/simplified coastlines. Fits terminal aesthetic MORE than the 160KB CC-BY-SA alternative and avoids attribution overhead. Committed.
- **Projection formula (equirectangular)**: `x = (lon + 180) / 360 * W`, `y = (90 - lat) / 180 * H`. Edge cases: dateline wraps to same x; poles distort infinitely (not an issue at 30° grid).
- **Marker design**: concentric rings — matches existing packet-flow/LED aesthetic in the chrome. 3 rings (outer/middle/inner-dot), 1px stroke, monochrome. Pulse animation optional.
- **Graticule**: dashed 30° grid, opacity ~0.4, monospace tick labels at edges ("30N", "150E", "0°").
- **Layout target**: 720×360 SVG viewBox (2:1 equirectangular). Aesthetic references: aviation charts (Jeppesen), CIC displays, Papers Please, Falcon 4 HSD.

## Ambiguity resolutions (from extract-map Section 4) — DECIDED

1. **LAYOUT+MAT apply loop split** → YES. Extract `applyPanelLayout(LAYOUT, MAT)` into `shared/chrome.js`. Per-page JS defines `LAYOUT` + `MAT` then calls the shared applier.
2. **`#big-rings`** → CHROME-SHARED. Same central rings on every page maintains the "one machine" feel.
3. **`.placard`** → CSS is CHROME-SHARED. HTML instances are PER-PAGE (each page picks its own placard text and positions).
4. **`spawnDecor()` content arrays** → engine shared, content arrays (crosses/glyphs/CODE) become a config argument per page. Main page keeps existing content; subpages may override.
5. **Glitch engine target list** → engine shared, expose `registerGlitchTargets([selectors...])` API. Main page registers its main-only targets (`#hexcol`, `#p-hex .pcaption`, etc). Subpages register their own.
6. **BOOT array** → MAIN-ONLY (per-page narrative). Passed as arg to shared loader engine's start function.
